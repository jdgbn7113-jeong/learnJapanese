# RimHahaha · 서비스 기획서

> 부수 분해로 외우는 한자 · 소세키의 「こころ」로 배우는 일본어
> 2026년 4월 기준 설계 문서

---

## 0. 서비스 한눈에 보기

### 0.1 제품 정의
일본어 학습자(특히 **JLPT 응시자**와 **원문 독해를 지향하는 중상급자**)를 위한 **단권형 통합 학습 웹앱**이다. 기존의 플래시카드식 앱이 가진 한계 — 문맥 없이 쪼개진 단어, 손으로 써보지 않고 지나치는 한자, 한자·어휘·문장이 서로 연결되지 않는 단절 — 을 해결하는 것을 목표로 한다.

### 0.2 핵심 가치 제안
| 가치                  | 구현 방식                                                                              |
| --------------------- | -------------------------------------------------------------------------------------- |
| **한자의 구조 이해**  | 단순 암기 대신 **부수(Radical) 분해 + 이야기형 니모닉(Mnemonic Story)** 으로 학습     |
| **손으로 쓰는 학습**  | Canvas 기반 **필기 연습**(한자 상세·단어 상세·문장 따라쓰기) — 연습/실전 모드 전환    |
| **맥락 있는 어휘**    | JLPT 레벨별 어휘 목록 ↔ 한자 상세 ↔ 소설 속 실제 용례가 **양방향 링크**로 연결됨      |
| **원문 독해의 사다리**| 루비(후리가나) → 히라가나 → 직역 → 의역 → 문학적 번역 **4단계 번역 스택**             |

### 0.3 왜 이 네 페이지인가 — 학습 루프 설계
```
[메인] 학습 동선 선택
   ├─▶ [한자] 부수·니모닉·필기로 '문자' 단위 내재화
   │        └─ (한자 상세 → 예시어) ──┐
   ├─▶ [단어] POS 분류 + 필기로 '어휘' 단위 내재화
   │        └─ (단어 상세 → 포함 한자) ──┤
   └─▶ [소설] '문장' 단위에서 위 둘을 실전 적용
            └─ (소설 단어 → 단어 상세)  ──┘
```
세 페이지는 독립된 기능이 아니라 **문자 → 어휘 → 문장**이라는 언어 단위의 층위를 그대로 따르는 학습 파이프라인이다. 메인은 이 파이프라인의 **진입점**이고, 각 상세 페이지의 "여기서 저기로" 이동 경로는 실제 학습자가 모르는 것을 만났을 때 취하는 자연스러운 행동 — "이 문장 속 이 단어가 뭐지?" → "이 단어에 들어간 이 한자는 뭐지?" — 을 그대로 디지털화한 것이다.

### 1.2 공통 셸(Shell) 구조
모든 페이지는 `Layout`으로 래핑되며 다음 3요소를 공유한다.

- **Sidebar** (데스크톱 / 모바일 드로어)
  - 브랜드 로고
  - **SearchBar** (한자 자동완성)
  - 네비게이션: 한자 · 단어 · 소설
  - 설정 버튼 (폰트 변경 모달)
- **Main** — 본문 영역
- **Footer** — 저작 정보

**폰트 설정 기능:** 선택 가능한 일본어 폰트 9종(명조·고딕·손글씨 계열)을 제공하고 `localStorage`에 저장한다. 학습자가 자신의 눈에 가장 잘 익는 서체로 공부하도록 하기 위함이며, 저시력·난독 사용자 접근성 고려도 포함한다.

## 2. 메인 페이지 (`/` — `HomePage.tsx`)

### 2.1 이 페이지가 존재해야 하는 이유
일본어 학습 서비스는 흔히 로그인 후 "진도" 대시보드로 직행한다. 본 서비스는 **진도 추적형이 아닌 참조형(reference-style)** 이므로, 메인은 **"오늘 무엇을 할지"를 3초 안에 결정**하도록 설계되었다. 사용자의 머릿속 질문은 세 가지뿐이다.
1. 찾으려는 한자가 있다 → **검색**
2. 체계적으로 레벨을 훑고 싶다 → **JLPT 한자 / 단어 목록**
3. 문맥 속에서 읽고 싶다 → **소설**

### 2.2 구성 요소
- 서비스 타이틀과 서브카피
- **SearchBar** (큰 변형 — 자동완성 드롭다운 포함)
- 3개의 진입 카드
  1. 소설로 일본어 배우기 (`/novel`)
  2. 한자 학습 JLPT N5~N1 (`/jlpt/n5`)
  3. 단어 학습 JLPT N5~N1 (`/vocab/n5`)
- JLPT 퀵 링크 버튼 5개 (N5, N4, N3, N2, N1)

### 2.3 설계 포인트
- **소설 진입 카드를 시각적으로 가장 돋보이게** 구성한다. 이유는 단순한 강조가 아니라, **본 서비스의 가장 차별적인 경험**(원문 독해)을 학습자에게 강하게 제시하기 위함이다. JLPT 암기 앱은 많지만 소설 독해 도구는 드물어 USP의 얼굴이 된다.
- **검색 바를 카드 위에 배치**한다. "이미 찾으려는 대상이 있는 사용자"를 가장 먼저 해결해주는 것이 리텐션에 중요하다. 카드 → 리스트 순서로 깊이가 늘어나므로 가장 얕은 진입부터 위에 둔다.
- **JLPT 퀵 링크(N5~N1)** 은 "카드 → 레벨 탭"의 2-click 과정을 **1-click**으로 줄인다. 반복적인 동선(이미 알고 있는 레벨로 진입)을 최단화.
- **상태 없음**: 메인은 순수 정적 페이지(data hook 없음). 로딩 지연 없이 즉시 렌더되어 체감 속도를 확보한다.

## 3. 한자 페이지 (Hanja)

한자 영역은 **목록**과 **상세** 두 페이지로 이원화된다. 학습자의 인지 부하를 기준으로 자른 것이다: 목록은 "훑어보기", 상세는 "파고들기".

### 3.1 한자 목록 (`/jlpt/:level` — `JlptListPage.tsx`)

#### 3.1.1 목적
JLPT 레벨 안에 속하는 모든 한자(예: N5 약 80자)를 **시각적으로 동시에 파악**할 수 있게 한다. 리스트가 아닌 **그리드**를 택한 이유는, 학습자가 레벨 전체를 "한 덩어리"로 기억하도록 돕기 위함이다. 언어 학습은 카테고리 전체의 감을 잡는 것이 개별 항목 암기 못지않게 중요하다.

#### 3.1.2 구성
- 타이틀 + 해당 레벨의 총 한자 수
- 레벨 탭 5개 (N5/N4/N3/N2/N1)
- 반응형 그리드 — 화면 크기에 따라 열 수가 늘어나, 넓은 화면에서는 레벨 전체를 한 스크롤 안에 볼 수 있음
- 각 KanjiCard: 한자 글자(대형), 의미(소형), 읽기(더 작게)
- 카드 클릭 시 `/kanji/:char`로 이동

#### 3.1.3 설계 포인트
- **반응형 컬럼 수**: 한자의 밀도가 화면 크기에 비례하도록 설계. 넓은 화면에서는 전체 레벨이 한 스크롤 안에 들어가도록 의도.
- **카드 내부의 정보 우선순위**: 글자 > 뜻 > 읽기 순으로 시각적 계층이 급격히 내려간다. **글자 자체**가 먼저 뇌에 꽂히고 메타정보는 부차적으로 인지되도록 했다.
- **레벨 전환이 가장 빈번한 동작**이므로 활성 탭 상태가 명확히 구분되어야 한다.

### 3.2 한자 상세 (`/kanji/:char` — `KanjiDetailPage.tsx`)

#### 3.2.1 목적 — 서비스의 코어
**이 페이지가 본 제품의 존재 이유이다.** 한자 암기의 실패는 대개 "읽기·뜻만 외우고 구조를 이해하지 못해서" 일어난다. 본 페이지는 하나의 한자를 다음 5단계로 해체·재조립한다.

1. **인식**: 큰 글씨로 글자를 제시
2. **분석**: 부수로 분해 (RadicalCombiner)
3. **서사화**: 각 부수에 인격을 부여하고 이야기로 연결 (MnemonicStory)
4. **필기**: 부수별·전체 글자별 Canvas 연습 (PracticeSlot × N)
5. **적용**: 해당 한자가 들어간 실제 단어 (ExampleWords)

#### 3.2.2 페이지 구성
- **헤더 블록**: 한자 글자, 의미, 음독/훈독, 태그(JLPT 레벨, 획수, 학년)
- **RadicalCombiner**: 부수들을 조립해 한자가 만들어지는 과정을 시각적 수식으로 표현
- **MnemonicStory**: 부수 페르소나 + 이야기 + 핵심 장면(keyImage)
- **필기 연습 영역**: 부수별 + 전체 한자 각각에 대한 연습 슬롯
- **ExampleWords**: 이 한자가 포함된 실제 단어 목록
- **이전/다음 한자 이동**: 같은 JLPT 레벨 내 순차 네비게이션 (데스크톱 / 모바일 각각 대응)

#### 3.2.3 핵심 컴포넌트의 설계 의도

##### RadicalCombiner — "부수 = 레고 블록" 은유
부수 박스를 나란히 배치하고 `+` → `=` → 한자 박스로 이어지는 수식 구조. **'조립된다'는 감각**을 시각적 은유로 제공한다. 본문과 구분된 독립 섹션으로 인지되도록 배치한다.

##### MnemonicStory — "부수에 페르소나 부여"
예: `休` = `人(사람)` + `木(나무)`
→ 이야기: "**사람이 나무에 기대어 쉰다**."
- `radicalRoles`: 각 부수에 캐릭터(페르소나)를 부여 — 단순히 "사람 인"이 아니라 "지친 여행자"
- `story`: 부수들의 상호작용을 1-2문장 서사로
- `keyImage`: 핵심 장면을 한 구절로 압축

**왜 이야기인가?** 인간의 기억은 의미 없는 기호(한자)보다 서사에 강하다(서사 부호화 이론). 부수가 이미 수천 년의 의미를 지닌 기호이므로, 각 부수에 역할을 주어 문장을 만들면 글자가 "장면"이 된다.

##### PracticeSlot (3-grid) — "보고 → 따라 쓰고 → 기억에서 꺼내 쓰고"
3칸으로 제공되는 것은 의도적이다.
1. **첫 칸**: 가이드 글자 위에 덧쓰기
2. **둘째 칸**: 희미한 가이드로 반 자동
3. **셋째 칸**: 가이드 없이 재현

캔버스에는 중앙 십자 + 대각선 가이드가 희미하게 그려져 한자 균형을 잡아준다. 손끝의 운동 기억(motor memory)은 눈으로만 외운 것보다 오래간다 — 이 페이지가 필기 기능을 포기할 수 없는 이유.

##### 이전/다음 네비게이션의 접근성
상세 페이지 체류 중 **가장 빈번한 행동이 "다음 한자로 이동"** 이다. 따라서 이동 버튼은 스크롤 위치와 무관하게 항상 접근 가능해야 한다. 장시간 학습 세션에서 "뒤로가기 → 리스트 → 다음 클릭"의 3단 동선이 축적되는 것을 막는 **동선 단축** 장치.

## 4. 단어 페이지 (Words)

### 4.1 단어 목록 (`/vocab/:level` — `VocabListPage.tsx`)

#### 4.1.1 목적
JLPT 레벨별 어휘 전체를 **품사(POS)로 필터링**하며 스캔한다. 단어 암기에서 품사 분류는 단순한 태그가 아니라 **문장에서의 행동 양식**을 결정하는 1차 정보다 — 동사인지 형용사인지 조사인지에 따라 활용법이 완전히 다르다.

#### 4.1.2 구성
- 타이틀 + 해당 레벨의 총 단어 수
- 레벨 탭 5개 (N5/N4/N3/N2/N1)
- **POS 필터 버튼**: "전체" + 해당 레벨에 존재하는 품사만 노출 (각 버튼에 개수 표기: `명사 (210)`)
- 단어 리스트: 각 행은 한자/가나, 의미, 품사 배지로 구성
- 행 클릭 시 `/vocab/:level/:index`로 이동

**지원 품사 13종:** 명사, 동사, い형용사, な형용사, 부사, 대명사, 의문사, 연체사, 수사, 조사, 접속사, 감탄사, 인사

#### 4.1.3 설계 포인트
- **POS 배지로 품사를 시각 구분**하면 학습자는 몇 차례 사용 후 "특정 배지 = 특정 품사"의 **시각-품사 연합**이 형성되어 리스트 스캔 속도가 빨라진다.
- **POS 옆 개수 표시**(`명사 (210)`): 해당 레벨의 품사 분포를 학습자가 암묵적으로 학습한다. N5는 명사·동사 비중이 크고, 고급으로 갈수록 조사·접속사가 늘어나는 일본어의 특성을 수치로 체감.
- **행의 3요소 배치**(한자/가나 | 의미 | POS): 시선이 좌→우로 흐르며 **"글자-의미-분류"** 순으로 인식되도록 한다.
- `useMemo`로 필터/카운트 메모이제이션. N1처럼 단어 수가 많은 레벨에서도 필터 전환이 즉각적.

### 4.2 단어 상세 (`/vocab/:level/:index` — `VocabDetailPage.tsx`)

#### 4.2.1 목적
단어 한 개를 **쓰기 + 구성 한자 학습**까지 파고든다. 단어 상세가 한자 상세와 반대 방향에서 쌍을 이룬다.
- 한자 상세: 글자 → 이 글자가 들어간 **단어 예시**
- 단어 상세: 단어 → 이 단어에 들어간 **한자 링크**

즉 두 상세 페이지는 **양방향 포털**이다.

#### 4.2.2 페이지 구성
- **헤더**: 단어 표기(한자/가나 대형 + 가나 읽기), 의미, 태그(JLPT 레벨, POS), 목록으로 복귀 링크
- **한자 배우기 섹션** (단어에 한자가 있는 경우): 단어에 포함된 각 한자 버튼 — 클릭 시 `/kanji/:char`로 이동. 각 한자의 대표 뜻 2개 미리보기.
- **필기 연습 영역**:
  - **연습 / 실전 모드 토글**
  - 가나 쓰기 캔버스
  - 한자+가나 쓰기 캔버스
- **이전/다음 단어 이동**: 같은 레벨 내 순차 네비게이션

#### 4.2.3 설계 포인트 — "연습 모드"의 의미
**`연습 / 실전` 토글**이 이 페이지의 백미다. 상태는 `localStorage`의 `vocab:practiceMode`에 저장된다.

- **연습 모드**: 캔버스 각 셀에 **흐린 가이드 글자**와 **메모 영역의 의미 힌트**가 노출. 학습자가 첫 노출에서 성공 경험을 얻도록.
- **실전 모드**: 모든 힌트 제거. 빈 셀만 — 학습자가 자기 기억에서 꺼내서 써야 함.

이는 인지심리학의 **회상(recall) > 재인(recognition)** 을 구조적으로 유도한다. 복습 사이클이 반복될수록 학습자가 스스로 실전으로 전환할 동기가 생긴다. 토글에는 `aria-pressed` 속성을 부여해 스크린 리더 접근성을 확보했다.

- **한자 배우기 섹션의 역할**: "이 단어를 더 깊이 배우려면 여길 눌러"의 초대장. 해당 한자를 누르면 `KanjiDetailPage`로 이동 — 학습 루프가 이 지점에서 한 차례 재귀된다.

## 5. 소설 페이지 (Novel)

소설 영역은 **목록(단락 인덱스) → 단락 내 문장 목록 → 문장 리더** 3단 깊이로 구성된다. 이는 책의 물리적 구조(장 → 페이지 → 문장)를 디지털로 그대로 옮긴 것으로, 학습자가 자신의 진도를 "목차의 어디에 있는지"로 인지하도록 돕는다.

### 5.1 소설 목록 (`/novel` — `NovelListPage.tsx`)

#### 5.1.1 목적
나쓰메 소세키의 대표작 「こころ」를 원문으로 읽는 입구. **한 스크린에 전체 단락 지도**를 제공해, 학습자가 "총 분량"과 "현재 공개된 분량"을 한눈에 파악한다.

#### 5.1.2 구성
- 헤더: "SOSEKI · こころ" / "소설로 배우는 일본어" / 작품 소개 카피
- 전체 단락 그리드 (57단):
  - **활성 카드**: "第 XX 段" 라벨 + 문장 수 캡션. 클릭 시 `/novel/p/:pid`로 이동
  - **비활성 카드**: "第 XX 段" 라벨 + "준비 중" 캡션. 클릭 불가

#### 5.1.3 설계 포인트
- **활성/비활성 카드의 구분**: "아직 준비 중인 단락"이 시각적으로 구별되지만 전체 총량은 가리지 않는다. "아, 아직 여기까지는 없구나"가 아니라 "전체는 57단이구나, 지금은 N단까지"라는 맥락을 유지.
- **문장 수를 캡션에 표기**(`25 문장`): 단락의 "무게"를 미리 알려줘, 오늘 공부할 분량을 학습자가 스스로 계획할 수 있다.
- **정적 콘텐츠는 정적으로**: 단락 맵은 `novel-paragraphs.json`에서 한 번 로드되어 전역 캐시된다. 재방문 시 즉시 렌더.

### 5.2 소설 단락 (`/novel/p/:pid` — `NovelParagraphPage.tsx`)

#### 5.2.1 목적
단락 내 **모든 문장을 연속 리스트**로 제시. 각 문장은 원문 + 한국어 의미가 한 카드에 요약되어, 학습자가 "어느 문장부터 들어갈지"를 선택할 수 있다.

#### 5.2.2 구성
- 목록으로 복귀 링크
- 헤더: "第 XX 段" / 문장 수
- 문장 리스트 — 각 항목은 순번(고정폭) + 원문 + 한국어 의미. 클릭 시 `/novel/:id`로 이동
- 하단 네비게이션: 이전 단락 / 다음 단락 링크 (있을 때만 노출)

#### 5.2.3 설계 포인트
- **문장 번호의 고정폭 정렬**: 고정폭 숫자로 세로 정렬하면 "아까 14번이 어디였지"를 빠르게 되찾을 수 있다. 소설 독해는 비선형 복귀가 잦다.
- **원문 + 의미의 2단 조판**: 원문이 시각적으로 우세하지만, 의미가 바로 아래 있어 한 번의 눈동자 이동으로 해석 완료. 이는 "원문만 읽겠다"와 "의미만 보겠다" 모두를 동시에 허용하는 조판.
- **하단 前段/次段 네비게이션**: 한 단락을 다 읽은 학습자가 바로 다음 단락으로 이어갈 수 있게 한다. 학습의 "탄력"을 끊지 않는다.

### 5.3 소설 리더 (`/novel/:id` — `NovelReaderPage.tsx`)

**본 서비스에서 정보 밀도가 가장 높은 페이지.** 하나의 문장을 6~8개의 층위로 해체한다.

#### 5.3.1 목적
단 한 문장을, 언어 학습이 필요한 **모든 관점**에서 분해 제시한다:
1. 시각적 인식 (원문 + 루비)
2. 음성적 인식 (히라가나 전사)
3. 의미 파악 (3단 번역: 직역/의역/문학)
4. 운동 기억 (문장 따라쓰기)
5. 어휘 분해 (어휘 표 + 딥다이브)
6. 문법 분해 (문법 포인트)
7. 서사 맥락 (앞·뒤·요약)
8. 문화·역사 주석

#### 5.3.2 페이지 구성 (순서)
1. 상단: 단락으로 돌아가기 링크 + "第 XX 段 · 第 YY 文 / ZZ" 문장 카운터
2. **원문 블록**: `<ruby>` 태그로 후리가나가 자동 삽입된 원문
3. **히라가나 섹션**: 전체 문장의 히라가나 전사 (강조 토큰 표시 지원)
4. **번역 스택**: 직역 / 의역 / (옵션)문학적 번역
5. **따라쓰기 캔버스** (SentenceTraceCanvas) — 접기/펼치기 가능
6. **한자 배우기 (어휘 표)**: 한자 | 발음 | 의미 | 단어 학습 링크
7. **어휘 심화 (vocabDeep)** — 선택적
8. **문법 포인트 (grammar)**
9. **맥락 (context)** — 선택적 (앞·뒤·요약)
10. **문화/역사 주석 (notes)** — 선택적
11. 이전/다음 문장 이동 (데스크톱 / 모바일 각각 대응)

#### 5.3.3 설계 포인트 — 왜 이렇게 많은 섹션인가

##### 번역의 3단 스택 — 학습자의 수준별 진입점
| 단계    | 의도                                     |
| ------- | ---------------------------------------- |
| 직역    | 구조 학습용. 조사·어순이 그대로 드러남   |
| 의역    | 의미 학습용. 자연스러운 한국어           |
| 문학    | (옵션) 문체 감상. 역자의 해석 포함       |

초급자는 직역부터, 중급자는 의역 위주로, 상급자는 문학적 번역과 원문을 비교. **같은 문장이 학습자의 수준에 따라 세 번 다르게 읽힌다.**

##### 루비(후리가나) 자동 삽입
`buildRubyPatterns()` + `parseOriginalWithFurigana()` 함수는 문장의 `vocab` 필드에서 (한자, 읽기) 쌍을 추출해 원문에 `<ruby>` 태그를 자동 삽입한다. **오쿠리가나(送り仮名)**, **중복/겹침**, **최장 일치 우선** 등의 엣지 케이스를 처리. 사람이 문장마다 수작업으로 후리가나를 달 필요가 없어 콘텐츠 확장이 쉽다.

##### 어휘 표의 하이퍼링크성
어휘 표의 각 행은 `vocab[].ref` 필드(`{ level, index }`)가 있으면 `/vocab/:level/:index`로 이동하는 링크가 된다. 즉 **소설 → 단어 상세 → (단어 상세 안의 한자 링크) → 한자 상세** 로 두 번의 클릭 안에 문장의 가장 깊은 원자까지 도달한다. 이것이 본 서비스가 이야기한 "양방향 포털"의 완성된 모습이다.

##### 따라쓰기 캔버스의 접기/펼치기
문장은 한자보다 훨씬 길어 캔버스가 화면을 크게 점유한다. 따라서 `SentenceTraceCanvas`는 **기본 접힘 + 클릭 시 펼침**이고, 이 상태는 `localStorage`에 저장된다. 한 번 펼친 사용자("나는 쓰면서 공부한다")는 이후 모든 문장에서 자동으로 펼쳐진 상태로 진입.

##### 맥락(context) 섹션 — 문장이 "소설 속 어디에 있는가"
```
{ before: "...", after: "...", summary: "이 문장은 ~의 시작을 알린다" }
```
개별 문장만 반복하면 "나무만 보고 숲을 못 보는" 독해가 된다. `context` 필드가 있는 문장에서는 전후 맥락과 서사적 역할을 제시해, **문장이 소설 구조 안에서 갖는 기능**을 학습자가 놓치지 않게 한다.

##### 상세 페이지 3형제의 공통 패턴
한자 상세 · 단어 상세 · 소설 리더는 모두 **이전/다음 순차 네비게이션**을 스크롤 위치와 무관하게 제공한다. 문장 간 이동이 학습자의 가장 빈번한 행동이므로, 긴 콘텐츠를 끝까지 스크롤해도 엄지손가락/마우스 동선이 일정해야 한다.

## 6. 설계 결정의 관통하는 원칙

### 6.1 "상세 페이지 3형제" 의 구조 통일
한자 상세 · 단어 상세 · 소설 리더는 모두 같은 뼈대를 쓴다.
- 헤더(대표 표기 + 메타데이터)
- 본문(해당 단위 고유의 학습 콘텐츠)
- 이전/다음 순차 네비게이션

이는 단순한 일관성이 아니라, 학습자가 **"상세 페이지에 들어왔다"는 맥락에서 취하는 공통 행동(=이전/다음으로 이동)** 의 동선 일관성을 확보하기 위함이다. 세 페이지의 인지 부담이 대칭이 된다.

### 6.2 캔버스의 보편성
필기 연습은 세 상세 페이지에 모두 등장한다.
- 한자 상세: `PracticeSlot × N`(부수별 + 전체)
- 단어 상세: `WordPracticeCanvas`(가나 행 + 한자+가나 행)
- 소설 리더: `SentenceTraceCanvas`(문장 전체)

입력 단위만 다를 뿐(글자/단어/문장), **"보고 끝내지 말고 써라"** 는 교육적 철학이 전 페이지에 관통한다.

### 6.3 정적 JSON + 클라이언트 캐시
모든 콘텐츠는 `public/data/*.json`으로 배포되고, 각 `useXxxData` 훅이 **모듈 레벨 전역 캐시**로 한 번만 로드한다. 서버/DB 없이도 빠르고, 오프라인 학습이 가능하며, 콘텐츠 수정은 JSON 파일 교체로 끝난다. `scripts/build-novel-index.mjs`는 원문 HTML을 파싱해 JSON을 생성하므로, **콘텐츠 제작 파이프라인이 독립**된다.

### 6.4 접근성 기본값
- 클릭 가능한 모든 요소는 `<Link>` 또는 `role="button" tabIndex={0}` + 키보드 핸들러
- 모드 토글은 `aria-pressed`
- 폰트 교체 기능으로 저시력·난독 사용자 및 글꼴 취향 차이 수용
- 로딩/에러/빈 상태 모두 텍스트로 명시

### 6.5 상태 최소화
- 서버 상태 없음 (외부 API / DB 없음)
- 클라이언트 상태는 `useState` + `localStorage`만 사용
- 저장되는 것은 **학습자 환경 설정**뿐: 폰트 선택(`kanji-master:font`), 단어 연습 모드(`vocab:practiceMode`), 문장 캔버스 펼침 상태
- 진도 추적은 의도적으로 **하지 않는다** — 참조형 서비스는 "어디서든 다시 들어와 꺼내보는" 도구이기 때문

## 7. 데이터 모델 요약

| 파일                       | 핵심 스키마                                                                    |
| -------------------------- | ------------------------------------------------------------------------------ |
| `kanji.json`               | `{ char, readings{onyomi,kunyomi}, meanings, jlpt, strokes, radicals[], mnemonic{radicalRoles,story,keyImage}, examples[] }` |
| `jlpt-index.json`          | `{ N5:[...], N4:[...], ..., N1:[...] }`                                        |
| `vocab-{n5..n1}.json`      | `{ level, words:[{kana, kanji, meaning, pos}] }` · POS 13종                    |
| `novel.json`               | `[{id, paragraph, original, hiragana[], meaning, vocab[ref?], grammar[], notes[], translations{literal,liberal,literary?}, vocabDeep?[], context?{before,after,summary}}]` |
| `novel-paragraphs.json`    | `{ totalParagraphs, paragraphs:[{paragraph, range:[start,end]}] }`             |

**데이터 간 링크 구조:**
```
novel[i].vocab[j].ref = { level, index } ──▶ vocab-{level}[index]
vocab-{level}[i].kanji  ──(글자별)──▶  kanji[char]
kanji[char].examples[j] ──(비정규 링크: 시각적 예시용)
```
즉 스토리지 수준에서 **문장 → 단어 → 한자** 의 계층이 명시적으로 걸려 있고, UI 는 이 링크를 그대로 "내비게이션"으로 노출한다.

## 8. 확장 로드맵 (참고용)

우선 순위가 높은 것부터:
1. **소설 콘텐츠 확장**: 현재 일부 단락만 활성화 → 「こころ」 전 57단 공개, 후속작(例: 「坊っちゃん」) 추가.
2. **사용자 진도 저장**: 현재 `localStorage` 기반 설정만 존재 — 마지막 본 한자/단어/문장 북마크, 학습 히트맵.
3. **필기 인식/채점**: 현재는 단순 드로잉 — 향후 stroke order 검증으로 확장.
4. **모바일 제스처**: 좌우 스와이프로 prev/next (현재의 버튼 기반 이동과 병행).
5. **검색 범위 확장**: 현재 한자 검색만 — 단어/소설 문장까지 통합 검색.

## 9. 페이지별 중점 요약표

| 페이지          | 한 문장 목적                                        | 중점(focus)                                       |
| --------------- | --------------------------------------------------- | ------------------------------------------------- |
| 메인            | 세 학습 동선의 진입을 3초 내 결정하게 한다          | **소설 카드 강조 + 검색 최우선**                  |
| 한자 목록       | JLPT 레벨의 한자 전체를 한 화면에서 동시 파악       | **반응형 그리드 + 레벨 탭**                       |
| 한자 상세       | 한자를 부수·이야기·필기로 **구조적으로** 내재화     | **부수 분해 + 니모닉 서사 + 필기 3-grid**         |
| 단어 목록       | 레벨별 어휘를 품사로 필터링하며 스캔                | **POS 필터 + 개수 표기**                          |
| 단어 상세       | 어휘를 쓰고, 포함 한자로 깊이 들어가는 포털         | **연습/실전 모드 토글 + 한자 상세 링크**          |
| 소설 목록       | 전체 장 구조와 진도 공개 상태를 파악                | **활성/비활성 카드 구분**                         |
| 소설 단락       | 단락 내 문장 목록에서 공부 시작점 선택              | **번호 정렬 + 원문/의미 2단 조판**                |
